# Planning Lessons

Operator-curated insights for the CORC planner. These are injected into every
planning session to guide task decomposition and spec development.

## Task Decomposition
- Keep tasks focused: one clear objective per task
- Prefer smaller tasks (1-3 checklist items) over large ones
- Always specify concrete, machine-testable done-when criteria

## Context Bundles
- Include only files the agent will actually need to read or modify
- Too many context files dilute attention; too few cause guessing

## Done-When Criteria
- Specific criteria (100+ chars) correlate with better outcomes
- Include test expectations: "tests pass" is better than "it works"
- Reference concrete artifacts: file paths, function names, CLI flags

## Roles
- Use scout for investigation before complex implementation
- Adversarial reviewers catch issues that implementers miss
